🏗️ User Guide - Golang Unit Test Generator

Complete guide for installing, configuring, and using the Golang Unit Test Generator Chrome extension.

🧩 1. Installation and Extension Access

Step 1: Enable Developer Mode

Open Chrome and make sure you have enabled Developer Mode.

Chrome Extensions Management Page

Step 2: Load Extension

  1. Go to: chrome://extensions/
  2. Click Load unpacked button → point to your extension folder (containing manifest.json, popup.html, etc.)

Loading unpacked extension

Step 3: Verify Installation

After successful installation, the extension icon will appear in the extension bar (top right of browser).

Extension icon in Chrome toolbar

🪄 2. Opening the Extension

  • Click the Golang Unit Test Generator icon to open the side panel (right side panel).
  • If the side panel doesn't automatically appear, make sure you have enabled sidePanel permissions in manifest.json.

Extension side panel interface

💻 3. Initial Interface

When the side panel opens, you will see:

👋 Welcome Message

Hi Programmer greeting with quick action buttons

🎯 Main Buttons

Generate Tests → to generate unit tests
Testing Guide → guide for running tests in Go
Project Setup → Go project structure explanation

🤖 Model Selection

Choose Model dropdown to select AI model (Mistral, Gemma, DeepSeek, etc.)

📂 4. Uploading or Inputting Go Code

Method 1: File Upload

  1. Click the 📁 (upload icon) button in the input area
  2. Upload a .go file from your local system

Method 2: Direct Input

  1. Copy your Go code directly
  2. Paste it into the text input box
// Example Go function to test
package main

import "fmt"

func Add(a, b int) int {
    return a + b
}

func main() {
    fmt.Println(Add(2, 3))
}

🤖 5. Generating Unit Tests

Step-by-step Process:

  1. Input Code: After entering your code (via upload or direct input)
  2. Select Model: Choose appropriate AI model from dropdown
  3. Send Request: Click the ➤ button to send prompt to the model
  4. Wait for Response: Wait for the model to generate unit test output
  5. Copy/Download: Use 📋 (copy) or 💾 (download) icons to save results

💡 Pro Tip

Use specific prompts like: "Generate comprehensive unit tests with table-driven tests and edge cases for this Go function"

🏗️ Go Project Structure Guide

Learn how to organize your Go projects for maximum efficiency and maintainability.

Standard Go Project Layout

myproject/
├── cmd/
│   └── myapp/
│       └── main.go
├── internal/
│   ├── handler/
│   │   ├── handler.go
│   │   └── handler_test.go
│   └── service/
│       ├── service.go
│       └── service_test.go
├── pkg/
│   └── utils/
│       ├── utils.go
│       └── utils_test.go
├── api/
│   └── openapi.yaml
├── web/
│   ├── static/
│   └── templates/
├── scripts/
├── docs/
├── go.mod
├── go.sum
├── Makefile
└── README.md

Directory Explanations

📁 cmd/

Main applications for this project. The directory name should match the name of the executable.

📁 internal/

Private application and library code. This is the code you don't want others importing.

📁 pkg/

Library code that's ok to use by external applications.

📁 api/

OpenAPI/Swagger specs, JSON schema files, protocol definition files.

📁 web/

Web application specific components: static web assets, server side templates.

Initialize New Project

# Create new directory
mkdir myproject && cd myproject

# Initialize Go module
go mod init github.com/username/myproject

# Create basic structure
mkdir -p cmd/myapp internal pkg api web docs scripts

# Create main.go
cat > cmd/myapp/main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
EOF

# Create go.mod if not exists
go mod tidy

Testing Structure

# Run all tests
go test ./...

# Run tests with coverage
go test -cover ./...

# Run tests in specific package
go test ./internal/handler

# Run specific test
go test -run TestHandlerCreate ./internal/handler

Makefile Example

.PHONY: build test clean run

build:
	go build -o bin/myapp cmd/myapp/main.go

test:
	go test -v ./...

coverage:
	go test -coverprofile=coverage.out ./...
	go tool cover -html=coverage.out

clean:
	rm -rf bin/

run:
	go run cmd/myapp/main.go

deps:
	go mod download
	go mod tidy
